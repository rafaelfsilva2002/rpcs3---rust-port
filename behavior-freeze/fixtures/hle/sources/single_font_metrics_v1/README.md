# single_font_metrics_v1 (R18 — cellFont byte-exact glyph metrics)

The first cellFont **rendering** slice: glyph **metrics** (no rasterization).
RPCS3 cellFont links `stb_truetype.h`; the Rust port uses the faithful
`stb_truetype` crate, so `cellFontGetCharGlyphMetrics` is **byte-exact**.

## Behaviour

A synthetic CC0 font (`font_data.h`, generated by `gen_font.py`: two rectangular
glyphs, unitsPerEm 1000, ascent 800, descent -200) is embedded and opened from
memory. Metrics for 'A' are checked against golden bit patterns at two scales:

```c
fontInit(cache>=24)                      // cellFontInitializeWithRevision (R17)
fontInitLibraryFreeType(&ftcfg, &lib)    // cellFontInitLibraryFreeTypeWithRevision
fontOpenFontMemory(lib, g_font_data, ..) // cellFontOpenFontMemory
fontSetScalePixel(&f, 1000, 1000)        // cellFontSetScalePixel -> scale 1.0
fontGetCharGlyphMetrics(&f, 'A', &m)     // cellFontGetCharGlyphMetrics
  // 'A' (100,0,500,700) adv 600: width=400, height=700, bearingX=100, advance=600
fontSetScalePixel(&f, 32, 32)            // scale = 32/1000
fontGetCharGlyphMetrics(&f, 'A', &m)
  // bit-exact: width=0x414ccccd, height=0x41b33334, bearingX=0x404ccccd, advance=0x4199999a
fontGetHorizontalLayout(&f, &hl)         // cellFontGetHorizontalLayout (scale 1.0)
  // baseLineY=800 (ascent), lineHeight=1000 (ascent-descent+lineGap), effectHeight=0 (lineGap)
return 0xC0DE;                            // iff every bit pattern matches
```

The metric math (cellFont.cpp:894-901): `width=(x1-x0)*scale`,
`scale = scale_y / (hheaAscent - hheaDescent)`, etc. — pure table reads + IEEE
single arithmetic, so it reproduces RPCS3 stb_truetype bit-for-bit.

## How it wires

- `cellFontInitLibraryFreeTypeWithRevision` (cellFontFT) → writes a non-null lib
  handle, CELL_OK (cellFontFT.cpp:8).
- `cellFontOpenFontMemory` (cellFont) → reads the font bytes from guest memory,
  parses with `StbttFont::open` (stb_truetype), stores host-side keyed by the
  CellFont guest address; writes fontdata_addr/origin (cellFont.cpp:110).
- `cellFontSetScalePixel` → writes scale_x/scale_y (FPR args) into the guest
  CellFont struct (cellFont.cpp:517).
- `cellFontGetCharGlyphMetrics` → looks up the parsed font, reads scale_y,
  computes the eight `CellFontGlyphMetrics` f32 fields, writes them BE
  (cellFont.cpp:868).
- `cellFontGetHorizontalLayout` → baseLineY/lineHeight/effectHeight from the font
  v-metrics (ascent/descent/lineGap) scaled by scale_y, written BE (cellFont.cpp:536).

## Result

`EmuCore::run_self` exit status = **0xC0DE**, vs a distinct `0xBADn` for each
failing step (open / scale / per-field metric mismatch).

## Calibration

`rust/rpcs3-hle-cellfont/tests/font_calibration.rs` independently asserts the
`stb_truetype` crate reproduces these golden values bit-for-bit — the byte-exact
contract with RPCS3 is test-enforced, not assumed.

## Deferred

Rasterization (`cellFontRenderCharGlyphImage`, glyph bitmaps) is a later slice.

## Consumed by

`rust/rpcs3-emu-core/tests/hle_font_metrics.rs`. The `.self`/`.elf` are built
locally via the PSL1GHT Docker toolchain and gitignored; `testfont.ttf` +
`font_data.h` are committed (CC0, regenerable via `gen_font.py`).
